Sum of Large Numbers题解

2020-05-11 题解

Sum of Large Numbers

标签： 题解

题面翻译

蒟蒻的我永远都是在这件事上花费了很长的时间

给两个数 NNN 和 KKK ，于是我们就有 N+1N + 1N+1 个整数 : 10100{10}^{100}10100 , 10100+1{10}^{100} + 110100+1 ， 10100+2{10}^{100} + 210100+2 , 10100+3{10}^{100} + 310100+3 ， ······ ， 10100+N{10}^{100} + N10100+N 。而我们每次可以从这 N+1N + 1N+1 个整数中取出至少 KKK 个整数，得到它们的和。问：一共可以得到多少种不同的和。

分析

想必各位dalao们都不会屑于写这种一眼题的题解，于是我就来凑数了
本蒟蒻在做这道题的时候，竟然错误地认为这是一道 O(1)O(1)O(1) 的题 QAQQAQQAQ ；
多亏一旁的朱某神提醒，看了一眼数据范围—— 1≤N≤2×1051 \leq N \leq 2 \times {10}^{5}1≤N≤2×105 很显然这是一道 O(n)O(n)O(n) 的题；
然后我们看到这 NNN 个整数都含有一个很大的基数 10100{10}^{100}10100 ，这就说明，任意 mmm 个整数的和会远远大于任意 m−1m - 1m−1 个整数的和，也就是它们之间绝对不会有重复；
不过，去相同的个数的整数之间的和是会有重复的，但显然，我们可以求出取出 mmm 个数的最大值和最小值，然后得到和的个数；
刚好，这就是一个 O(n)O(n)O(n) 的算法，于是我们就可以开始快乐的打代码了。

代码核心

从 KKK 到 N+1N + 1N+1 循环，然后通过“等差数列求和”写出每个 mmm 所对应的和的个数，然后取模。

for(int i = m; i < n; i ++) {
		ans += ((n + (n - i + 1)) * i) / 2 - (((i - 1) * i) / 2) + 1;
		ans %= mod;
	}

完整 ACACAC 代码

#include<bits/stdc++.h>
#define int long long

using namespace std;

const int maxn = 200010, mod = 1000000007;
int n, m;
int tem, cnt[maxn] = {0};

int read()
{
  int x = 0, w = 1;
  char ch = 0;
  while (ch < '0' || ch > '9') {
    if (ch == '-') w = -1;
    ch = getchar();
  }
  while (ch >= '0' && ch <= '9') {
    x = x * 10 + (ch - '0');
    ch = getchar();
  }
  return x * w;
}

inline void write(int x)
{
  static int sta[35];
  int top = 0;
  do {
    sta[top++] = x % 10, x /= 10;
  } while (x);
  while (top) putchar(sta[-- top] + 48);
}

signed main()
{
	n = read();
	m = read();
	if(n + 1 == m) {
		cout<<1;
		return 0;
	}
	int ans = n + 2;
	for(int i = m; i < n; i ++) {
		ans += ((n + (n - i + 1)) * i) / 2 - (((i - 1) * i) / 2) + 1;
		ans %= mod;
	}
	write(ans);
	return 0;
}
